Detailed Description

An object that can connect to, and operate remote or fake devices. I have three general API groups:

  • connection: NTV2Connect, IsConnected, NTV2Disconnect;
  • configuration: ConnectParams, HasConnectParam, ConnectParam, ConnectHasScheme, SetConnectParams
  • inquiry: Name, HostName, Print, Version, etc.;
  • device operation: ReadRegister, WriteRegister, (classic) AutoCirculate, WaitForInterrupt, DMATransfer, NTV2Message.
  • The Create factory method attempts to find and load a plugin that implements the requested software or remote device.
    Note
    The API changed in SDK 16.3 to have most functions return bool (success/fail) instead of int. Check the AJADebug log for AJA_DebugUnit_RPCServer or AJA_DebugUnit_RPCClient messages.

◆ NTV2QueryDevices()

virtual bool NTV2RPCClientAPI::NTV2QueryDevices ( NTV2StringList &  outDeviceInfos,
std::string &  outErrMsg 
)
inlinevirtual

Queries the devices that are accessible on the remote host.

Parameters
[out]outDeviceInfosReceives a list of zero or more device information strings, one string per device. Each string is ':' delimited with the following information fields, in this order: "host:port:deviceid:serial:index" where...
  • host: server IPv4 dotted-quad address or host name;
  • port: server port number (decimal, no leading zeroes);
  • deviceID: NTV2DeviceID as 8-digit hex string (or device name);
  • serial: device serial number as 16-digit hex string, or its character string equivalent;
  • index: 16-bit unsigned index number (optional; only specified for real, connected hardware) Subclasses must re-implement to return whateever is appropriate.
[out]outErrMsgAn error message if NTV2QueryDevices is unsuccessful.
Returns
True if successful; otherwise false.
